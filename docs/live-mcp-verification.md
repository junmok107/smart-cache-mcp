# Smart Cache MCP — 실사용 연동 검증 기록

`benchmark/`의 자동화된 통계 검증과 별개로, 실제 MCP 클라이언트(Claude)에 붙여서 라이브로 도구를 호출해본 검증 기록이다. 개발 중 임시 스크립트로 확인한 것과 달리, 사용자가 실제로 쓰게 될 경로(Claude ↔ `mcp-remote` ↔ Smart Cache MCP SSE 엔드포인트)를 그대로 통해서 확인했다.

날짜: 2026-07-14

> 이 기록은 재랭킹(retrieve-then-rerank, CLAUDE.md 11장) 도입 이전 시점의 검증이다. 여기 나오는 "유사도"/"임계값(0.90)"은 당시의 단일 단계 코사인 유사도 기준이며, 현재는 코사인 top-K 검색(1단계) 이후 cross-encoder 재랭킹 점수(2단계, 기본 임계값 0.60)가 히트/미스를 결정한다 — 코사인 임계값 0.90은 재랭킹 서비스 장애 시 폴백 판정에만 쓰인다.

## 연동 방법

Claude 앱 설정의 `mcpServers`에 `mcp-remote` 브릿지로 등록:

```json
"smart-cache-mcp": {
  "command": "cmd",
  "args": ["/c", "npx", "-y", "mcp-remote", "http://localhost:3000/sse"]
}
```

연결 후 `listTools`로 5개 도구(`cached_call`, `register_mcp`, `cache_stats`, `cache_clear`, `cache_config`) 전부 정상 노출 확인. `cache_stats`를 최초 호출해 연결 자체를 확인했다.

## 테스트 방법

로컬에 `benchmark/downstream-server.ts`(8개 주제에 대해 결정론적 답변을 주는 지식조회 목업)를 독립 실행해 `register_mcp`로 등록한 뒤, `cached_call`을 실제 Claude 세션에서 직접 호출해가며 검증했다. 개발/기획/개발공부 세 가지 실사용 페르소나를 상정해 호출 패턴을 다르게 구성했다.

## 핵심 발견

- **오탐(false positive) 없음**: 개념적으로 인접한 주제(머신러닝 vs 딥러닝, 도커 vs 쿠버네티스)를 물어봐도 서로 다른 캐시 엔트리로 정확히 구분됐다. 이번 검증에서 가장 중요하게 확인하려던 지점.
- **문맥이 섞이면 유사도가 떨어진다**: 같은 질문이라도 비즈니스 맥락이나 "공부 중인데" 같은 부가 설명이 붙으면 유사도가 낮아져 임계값(0.90) 근처에서 미스로 밀리는 경우가 있었다 (예: #11 유사도 0.908, 아슬아슬하게 히트).
- **교차언어 매칭은 불완전**: "What is artificial intelligence?"는 이미 캐시된 한국어 "인공지능이란 무엇인가요?"와 매칭되지 않았다. 완벽한 교차언어 히트를 보장하지 않는다는 게 실사용에서도 재확인됨 (`benchmark/REPORT.md`의 다국어 시나리오 결과와 일치).
- **fallback 응답도 정상 캐싱**: 지식베이스에 없는 질문("정보를 찾을 수 없습니다" 응답)도 다른 응답과 동일하게 캐싱되고, 서로 다른 unknown 질문끼리는 오탐 없이 구분됐다.
- **자체 신고 통계 vs 수동 카운트 완전 일치**: `cache_stats`가 보고하는 `total_entries`/`total_lookups`/`total_hits`가 두 라운드 모두 수동으로 센 값과 오차 없이 정확히 일치했다.
- **에러 처리 검증**: 미등록 별칭, 잘못된 JSON 모두 서버가 죽지 않고 명확한 에러 메시지를 반환했다.

## 정리

검증에 사용한 캐시 엔트리(15건), 누적 조회 로그(167건), 테스트용 `mcp_registry` 별칭(`live_test_kb`), 로컬 다운스트림 목업 서버 프로세스는 모두 검증 후 삭제/종료했다. 이 문서 자체가 검증 결과 기록이다.
